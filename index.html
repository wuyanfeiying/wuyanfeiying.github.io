【react】简述NodeJs,npm,Webpack,Es6和Babel

发表于 2020-07-11 分类于 react ， 读书

Node.js与npm

  • npm 是基于Node.js框架的包管理工具
  • Node.js框架则是基于著名的Google V8引擎所开发的JavaScript服务器端平台，可以用来快速搭建易于拓展的Web应用。

Webpack模块打包器

Webpack是一项模块打包器技术，通过分析项目结构以及其他一些浏览器不能直接运行的扩展语言或资源，将其转换并打包为合适的格式以供浏览器解析。

工作方式：

  1. 将项目作为一个整体，通过一个给定的主文件或入口文件（如index.js）
  2. Webpake将从这个文件开始查找项目的所有依赖文件
  3. 使用加载器（loaders）进行处理
  4. 最终打包成一个（或多个）可通过浏览器解析的JavaScript脚本文件

ES6和Babel

ES6:

  1. ES6 就是 ECMAScript 6 的缩写
  2. 是在JavaScript逐渐成熟并且成为事实上的脚本语言之后，经由Ecma国际标准化（欧洲计算机制造商协会前身）的脚本语言。
  3. ES6是一次重大的改革，正式发布2015年，因此ES6也称为ECMAScript 2015。

Babel:

  1. 是一个工具链
  2. 主要用于将基于ECMAScript 2015+标准编写的脚本代码转换为向后兼容的JavaScript语法的脚本代码
  3. 以便能够运行在当前或旧版本的浏览器中
  4. Babel可以当做下一代JavaScript脚本语言的编译器。

【react】初识(概述/优势/hello word)

发表于 2020-07-11 分类于 react ， 读书

概述

React 框架最初是社交网络巨头Facebook（脸书）公司的一个内部项目，设计目标是为了来构架Instagram网站的。Instagram在2012年被Facebook收购，在考虑给Instagram设计UI的时候，当时市面上找不到合适的UI框架，于是Facebook打算单独去研发。因此，React框架的设计思想很独特、视角很新奇，是很有革命的一款产品

react优势

React框架的设计初衷就是为了构建UI，而构建UI的核心思想就是封装组件。组件维护自身的状态和UI，每当状态发生改变的时候，就会自动重新渲染组件自身，而不需要通过反复的查找DOM元素后，再重新渲染整个组件了。

  1. 声明式设计：React采用声明范式，可以轻松描述应用。
  2. 高效：React 通过对DOM的模拟，最大限度地减少与DOM的交互。
  3. 灵活：React 可以与已知的库或框架很好地配合。
  4. 使用JSX语法：JSX是JavaScirpt语法的扩展，可以极大地提升JS运行效率。
  5. 组件复用：通过React 构建组件使得代码易于复用，可在大型项目应用开发中发挥优势。
  6. 单向响应的数据流：React 实现了单向响应的数据流，减少了重复代码，比传统数据绑定方式更简单。

练习demo

  • react.js 是react框架的核心库
  • react-dom.js提供与DOM相关的功能
  • bable.min.js将ES6转为ES5代码，这样在不支持ES6的浏览器上执行React代码也可以使用

    2. container参数：必需，标识渲染的目标对象（元素/组件）

框架设计模式（MVC/MVP/MVVM）

发表于 2020-07-06 更新于 2020-07-07 分类于 设计模式

设计模式（MVC/MVP/MVVM）的区别：

MVC（Model View Controller):

  • M: 数据保存
  • V：用户界面
  • C：业务逻辑

这种设计模式在后端用的较多，例如php框架等。
特点：

  1. 用户可以向view发送指令（dom事件），再由V直接要求M改变状态；
  2. 用户可以直接向C发送指令，再由C发送给V；
  3. C非常薄弱，只起到了路由的作用，而V特别厚重，业务逻辑都部署在了V。

缺点：
会导致页面加载特别慢。

MVP（Model View Presenter ):

  • M：数据保存
  • V：用户界面
  • P：改变了MVC中的设计方向
    特点：
  1. 各部分通讯都是双向的；
  2. V和M不再发生联系，都是通过P传递；
  3. V非常薄，没有任何业务逻辑（被动视图）；P非常厚，业务逻辑太多。

MVVM（ Model-View-ViewModel ）：

和MVP类似，将P换成了VM
特点：

  • 采用了双向数据绑定，V变动反映在VM上，反之亦然。

数据结构之数组

使用构造函数创建数组

  • 使用构造函数的方式创建数组,不需要传任何参数,就可以得到一个空数组.
  • 如果给数组传递参数,就可以得到有参数的数组.
  • 有些特殊的场景,可能会用到:”给定特定长度,填充固定的内容的数组”
  • 这个时候可以使用fill来进行实现

数组的访问和遍历

可以直接通过数组的索引值访问数组的元素

遍历数组:

  1. for循环
  2. forEach方法
    这是个回调函数,通过取forEach返回的第一参数和第二参数,来获取数组的每个元素值和索引
  3. map方法
  • 这个也是个回调函数,但是会返回一个新的数组,会对数组中的每个元素进行处理,最终返回一个全新的数组.
  • map方法不仅仅是对数组的遍历,更像是’再加工’,当我们需要对数组中的元素进行 批量修改且同时修改的逻辑又一致的时候,可以使用mao来达到目的.

从性能上来说, for 循环是遍历起来最快的.

二维数组

  • 一维数组是一条”线”布局,二维数组是一个”面”布局.
  • 在数学中,形如长方阵列排列的复数或实数集合,被称为”矩阵”.因此二维数组的别名就叫”矩阵”.

初始化二维数组

使用 fill 方法的局限性

使用fill方法看似创建了一个二维数组,但是当改变其中一个元素值的时候,会发现问题.

虽然只想改变,第一个元素数组的值,但是此刻,所有元素数组的值都发生了改变.

  • 原因: 当给 fill 传值的时候,如果入参的类型是引用类型,那么fill填充的时候其实填充的是引用类型.
  • 也就是说,用fill初始化二维数组的时候,看似给每个位置初始化了一个数组,其实7个数组对应了同一个引用,指向的是同一块内存空间
  • 所以,当想改变第0行第0个元素的时候,其实第1-6行的元素也会跟着改变.

初始化一个二维数组的正确方式

使用for循环,速度快,每一次迭代的时候,使用[]数组来代替新的元素,这样便不会有引用问题带来的尴尬.

二维数组的访问

一维数组需要for循环遍历一次,二维数组需要for循环遍历两次,N维数组则需要for循环遍历N次

盒模型

发表于 2020-06-27 分类于 css

css的盒模型由content(内容)、padding(内边距)、border(边框)、margin(外边距)组成。但盒子的大小由content+padding+border这几部分决定。

分类：盒模型分为 IE盒模型 和 W3C 标准盒模型
区别：二者的宽度和高度计算规则不一样。

  1. IE 盒模型 width = 内容的宽度 + padding + borderheight = 内容的高度 + padding + border
  2. W3C 标准盒模型 width = 内容的宽度height = 内容的高度

使用：在ie8+浏览器中使用哪个盒模型可以由box-sizing(CSS新增的属性)控制，默认值为content-box，即标准盒模型；如果将box-sizing设为border-box则用的是IE盒模型。如果在ie6,7,8中DOCTYPE缺失会触发IE模式。在当前W3C标准中盒模型是可以通过box-sizing自由的进行切换的。
