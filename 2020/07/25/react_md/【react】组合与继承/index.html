【react】组合与继承

发表于 2020-07-25 分类于 react

React框架在组件代码重用方便,具有组合模式与继承模式.React官方推荐设计人员在实际项目中尽量使用组合模式.

概述

现实中比较复杂的业务组件,考虑到代码质量问题,建议还是要通过组合模式来实现,这样可以避免设计人员在研究子组件时还要被迫去研究父组件.

定义组件容器

React建议先定义一个对话框通用组件(Box)组件,然后使用一个特殊的Prop属性(children)将未来的子组件传递到该容器中渲染,从而实现组件组合模式的设计.

例:

  • 定义一个对话框组函数组件MyDialogBox

  • 通过props属性参数传递进来对话框边框颜色属性color

  • 通过props属性参数children传递对话框子组件的内容(未定义的位置内容)

    定义子组件

    此处设计一个对话框子组件(WelcomeDialog),该子组件与组件容易(MyDialogBox),构成组合模式(实际是包含关系).

  • 需要注意点:

    1. props是父组件向子组件的传值,此处props.color是在父组件<MyDialogBox color="gray">定义的属性,子组件取出来直接渲染
    2. props.children,比较特殊,是父组件里面的内容,在子组件接受的时候会挂载到children中.

使用React组件的组合模式的优势: 可以基于一个组件容器和不同的子组件,构建出多种组件应用.

自定义Props属性

React组件组合模式除了可以使用prop参数属性(children)之外,还支持自行约定prop参数属性。这种自行约定方式摆脱了children的约束，使用起来更加灵活，只需要将所需内容传入prop参数，并使用相对应的属性。

例：

  • 定义了三个面板组件，放到了主面板容器中：
  • MainPanel // 主面板容器
    1. HeaderPanel // 头部
    2. LeftPanel // 左侧栏
    3. RightPanel // 右侧栏
  • 定义了组件入口APP,引入了主面板容器MainPanel
