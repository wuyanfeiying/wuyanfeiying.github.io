【react】React表单元素

发表于 2020-07-25 分类于 react

React表单中的DOM元素与React框架中的其他DOM元素有所不同,因为表单元素需要保留一些内部状态(State).

表单受控组件

  • 传统的HTML表单中的元素(如文本输入框<input>,下拉菜单<select>合入按钮<button>,等等)通常是自己维护自己的”状态”,根据用户输入的改变进行”状态”更新.
  • React框架下表单元素的可变状态(mutable state)通常保存在组件的state状态中.

React表单的可变状态时需要保存在组件的state状态属性中的,并且只能通过setState()方法进行维护更新.

处理多个输入

在React表单中,如果需要处理多个<input>标签元素,可以通过给每个元素添加各自name属性,并通过同一个事件处理方法进行处理.至于如何在同一个事件处理方法中区分不同的<input>标签元素,则是根据传递进来Event对象的name属性进行操作的.

受控组件与非受控组件

  • 受控组件: 通过State(状态)来维护组件,组件数据的更新时通过”自顶而下”的数据流方式来操作的.
  • 非受控组件: 通过与State(状态)无关的Refs概念实现.对于非受控组件,是允许组件直接操作事件DOM的.

非受控组件主要适用的几个场景:

  1. 管理焦点,文本选择或媒体播放.
  2. 触发强制动画.
  3. 集成第三方DOM库.

应避免过多的使用Refs直接操作DOM
